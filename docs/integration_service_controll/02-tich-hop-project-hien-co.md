# Tích hợp project đang host đơn lẻ vào Service Controller

> Hướng dẫn chuyển đổi các project đang chạy infrastructure riêng sang sử dụng shared infrastructure.

---

## 1. Tổng quan

Hiện tại mỗi project có thể đang tự khai báo infrastructure trong `docker-compose.yml` của mình:

```yaml
# TRƯỚC: project/docker-compose.yml có infra riêng
services:
  app:
    build: .
    ports:
      - "8000:8000"
    depends_on:
      - postgres
      - redis

  postgres:           # <-- Infrastructure riêng
    image: postgres:16
    ports:
      - "5432:5432"

  redis:              # <-- Infrastructure riêng
    image: redis:7
    ports:
      - "6379:6379"
```

Vấn đề:
- Nhiều project đều chiếm port 5432, 6379 → không thể chạy đồng thời
- Mỗi project tốn RAM riêng cho infra
- Backup/quản lý phân tán

Mục tiêu: Chuyển sang dùng shared infrastructure, chỉ giữ lại app code trong docker-compose.

---

## 2. Quy trình tích hợp

### Bước 1: Đảm bảo Service Controller đã sẵn sàng

```bash
# Cài đặt sc (nếu chưa)
cd C:\Projects\service_controller
pip install -e .

# Khởi tạo (nếu chưa)
sc init --workspace C:\Projects

# Đảm bảo infra đang chạy
sc infra up
```

### Bước 2: Đăng ký project với controller

```bash
sc project add C:\Projects\MyExistingProject --name my_project
```

Ghi lại output — controller sẽ thông báo namespace được cấp phát:

```
Registered project: my_project
  Path: C:\Projects\MyExistingProject
  Namespaces:
    postgres: my_project_db
    redis: 3
    minio: my_project-bucket
    milvus: my_project_
    elasticsearch: my_project_
  Port range: 8030-8039
```

### Bước 3: Migrate database (nếu có data)

Nếu project đã có data trong PostgreSQL riêng, cần migrate sang shared instance.

#### 3a. Export data từ postgres cũ

```bash
# Giữ project cũ để đảm bảo postgres cũ còn chạy
cd C:\Projects\MyExistingProject
docker compose up -d postgres

# Export toàn bộ database
docker exec -it <old-postgres-container> pg_dump -U postgres -d <old_db_name> > backup.sql
```

#### 3b. Import vào shared postgres

```bash
# Đảm bảo shared infra đang chạy
sc infra up

# Database đã được tạo tự động khi đăng ký project
# Import data
docker exec -i infra-postgres psql -U postgres -d my_project_db < backup.sql
```

#### 3c. Migrate Redis data (nếu cần)

Redis data thường là cache, có thể bỏ qua. Nếu cần migrate:

```bash
# Export từ Redis cũ
docker exec <old-redis-container> redis-cli DUMP <key> > key.dump

# Import vào shared Redis với đúng DB number
docker exec infra-redis redis-cli -n 3 RESTORE <key> 0 "$(cat key.dump)"
```

> Gợi ý: Với Redis, thường chỉ cần để app tự tạo lại cache khi khởi động.

#### 3d. Migrate MinIO data (nếu cần)

```bash
# Cài mc (MinIO Client) nếu chưa có
# Cấu hình endpoint cũ và mới
mc alias set old-minio http://localhost:9000 minioadmin minioadmin
mc alias set shared-minio http://localhost:9000 minioadmin minioadmin

# Tạo bucket mới
mc mb shared-minio/my_project-bucket

# Copy data
mc mirror old-minio/old-bucket shared-minio/my_project-bucket
```

### Bước 4: Sửa docker-compose.yml của project

**Xóa tất cả infrastructure services**, chỉ giữ lại app code. Thêm `infra-net` external network.

```yaml
# SAU: project/docker-compose.yml — chỉ có app
services:
  app:
    build: .
    container_name: my-project-app
    ports:
      - "8030:8000"          # Dùng port từ allocated range
    env_file: .env
    deploy:
      resources:
        limits:
          memory: 2048M
          cpus: "1.0"
        reservations:
          memory: 256M
          cpus: "0.25"
    networks:
      - infra-net

  # Nếu có worker/celery/consumer...
  worker:
    build: .
    container_name: my-project-worker
    command: celery -A app worker
    env_file: .env
    deploy:
      resources:
        limits:
          memory: 1024M
          cpus: "0.5"
    networks:
      - infra-net

# KHÔNG khai báo volumes cho infra (pgdata, redisdata...)

networks:
  infra-net:
    external: true           # QUAN TRỌNG: join vào shared network
```

**Lưu ý quan trọng**:
- **`networks: infra-net: external: true`** — bắt buộc để kết nối vào shared infra
- **Port mapping** — dùng port trong range được cấp phát (vd: 8030-8039), tránh conflict với project khác
- **Xóa `depends_on`** cho infra services (postgres, redis...) — chúng không còn trong cùng compose file

### Bước 5: Tạo file .env mới

```bash
sc project env my_project --output C:\Projects\MyExistingProject\.env
```

File `.env` sẽ chứa:

```env
# Auto-generated by service-controller for: my_project

DATABASE_URL=postgresql://postgres:postgres@infra-postgres:5432/my_project_db
REDIS_URL=redis://infra-redis:6379/3
MILVUS_HOST=infra-milvus
MILVUS_PORT=19530
MILVUS_COLLECTION_PREFIX=my_project_
ELASTICSEARCH_URL=http://infra-elasticsearch:9200
ELASTICSEARCH_INDEX_PREFIX=my_project_
MINIO_ENDPOINT=infra-minio:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=my_project-bucket
```

### Bước 6: Cập nhật code đọc env variables

Kiểm tra code của project đảm bảo đọc connection strings từ environment variables, không hardcode.

**Python (SQLAlchemy):**

```python
import os
from sqlalchemy import create_engine

# TRƯỚC (hardcode)
# engine = create_engine("postgresql://postgres:postgres@localhost:5432/mydb")

# SAU (đọc từ env)
engine = create_engine(
    os.getenv("DATABASE_URL"),
    pool_size=15,
    max_overflow=10,     # Tổng max 25, trong limit 40/project
    pool_timeout=30,
    pool_recycle=1800,
)
```

**Python (Redis):**

```python
import os
import redis

# TRƯỚC
# r = redis.Redis(host="localhost", port=6379, db=0)

# SAU
r = redis.from_url(os.getenv("REDIS_URL"))
```

**Python (Milvus):**

```python
import os
from pymilvus import connections

# TRƯỚC
# connections.connect(host="localhost", port="19530")

# SAU
connections.connect(
    host=os.getenv("MILVUS_HOST"),
    port=os.getenv("MILVUS_PORT"),
)
prefix = os.getenv("MILVUS_COLLECTION_PREFIX")
collection_name = f"{prefix}documents"  # vd: my_project_documents
```

**Python (Elasticsearch):**

```python
import os
from elasticsearch import Elasticsearch

# TRƯỚC
# es = Elasticsearch("http://localhost:9200")

# SAU
es = Elasticsearch(os.getenv("ELASTICSEARCH_URL"))
index_prefix = os.getenv("ELASTICSEARCH_INDEX_PREFIX")
index_name = f"{index_prefix}documents"  # vd: my_project_documents
```

**Python (MinIO):**

```python
import os
from minio import Minio

# TRƯỚC
# client = Minio("localhost:9000", ...)

# SAU
client = Minio(
    os.getenv("MINIO_ENDPOINT"),
    access_key=os.getenv("MINIO_ACCESS_KEY"),
    secret_key=os.getenv("MINIO_SECRET_KEY"),
    secure=False,
)
bucket = os.getenv("MINIO_BUCKET")
```

### Bước 7: Dừng project cũ, khởi động project mới

```bash
# Dừng project cũ (bao gồm infra riêng)
cd C:\Projects\MyExistingProject
docker compose down

# Khởi động project mới (chỉ app, dùng shared infra)
sc project up my_project
```

### Bước 8: Kiểm tra

```bash
# Xem trạng thái tổng thể
sc status

# Kiểm tra app có kết nối được không
docker logs my-project-app

# Kiểm tra database
docker exec -it infra-postgres psql -U postgres -d my_project_db -c "\dt"

# Kiểm tra Redis
docker exec infra-redis redis-cli -n 3 KEYS "*"
```

## 3. Xử lý các trường hợp đặc biệt

### 3.1 Project dùng nhiều database

Nếu project cần nhiều database (vd: 1 cho app, 1 cho Langfuse), tạo thêm thủ công:

```bash
docker exec -it infra-postgres psql -U postgres -c "CREATE DATABASE my_project_langfuse_db;"
```

Thêm DATABASE_URL thứ 2 vào `.env`:

```env
DATABASE_URL=postgresql://postgres:postgres@infra-postgres:5432/my_project_db
LANGFUSE_DATABASE_URL=postgresql://postgres:postgres@infra-postgres:5432/my_project_langfuse_db
```

### 3.2 Project có custom PostgreSQL extensions

Shared PostgreSQL đã có pgvector. Nếu cần extension khác:

```bash
docker exec -it infra-postgres psql -U postgres -d my_project_db -c "CREATE EXTENSION IF NOT EXISTS <extension_name>;"
```

### 3.3 Project dùng custom Redis config

Redis shared đã được cấu hình với `maxmemory 768mb` và `allkeys-lru`. Nếu project cần config khác (vd: specific eviction policy), cần dùng chung config hoặc thêm Redis instance riêng qua `sc infra add`.

### 3.4 Project cần kết nối từ host (dev không dùng Docker)

Khi chạy app trực tiếp trên máy (không trong container), dùng `localhost` thay vì container name:

```env
# .env cho dev không Docker
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/my_project_db
REDIS_URL=redis://localhost:6379/3
MILVUS_HOST=localhost
MILVUS_PORT=19530
ELASTICSEARCH_URL=http://localhost:9200
MINIO_ENDPOINT=localhost:9000
```

### 3.5 Project không dùng tất cả services

Không bắt buộc phải dùng tất cả services. Nếu project chỉ cần PostgreSQL và Redis, chỉ cần dùng 2 env variable tương ứng. Các namespace khác (Milvus, ES, MinIO) được cấp phát nhưng không tạo ra resource cho đến khi app thực sự dùng.

## 4. Checklist tích hợp

- [ ] Service Controller đã cài đặt và `sc init` đã chạy
- [ ] Project đã đăng ký: `sc project add <path>`
- [ ] Data đã migrate (nếu có): PostgreSQL, Redis, MinIO
- [ ] `docker-compose.yml` đã xóa tất cả infra services
- [ ] `docker-compose.yml` đã thêm `infra-net: external: true`
- [ ] Port mapping dùng range được cấp phát
- [ ] File `.env` đã tạo: `sc project env <name> -o .env`
- [ ] Code đọc connection từ env variables, không hardcode
- [ ] Collection/index dùng prefix được cấp phát
- [ ] Bucket name dùng tên được cấp phát
- [ ] Test: `sc project up <name>` thành công
- [ ] Test: App kết nối được tất cả services cần thiết
- [ ] Dọn dẹp: Xóa Docker volumes cũ (sau khi đảm bảo data đã migrate xong)

## 5. Ví dụ thực tế: Tích hợp project Rag_Service

### Trước khi tích hợp

```
Rag_Service/
├── docker-compose.yml     # Chứa: app + postgres + redis + milvus
├── .env                   # Connection tới localhost
└── src/
```

`docker-compose.yml` cũ:

```yaml
services:
  app:
    build: .
    ports: ["8000:8000"]
    depends_on: [postgres, redis, milvus]

  postgres:
    image: postgres:16
    ports: ["5432:5432"]
    volumes: [pgdata:/var/lib/postgresql/data]

  redis:
    image: redis:7
    ports: ["6379:6379"]

  milvus:
    image: milvusdb/milvus:v2.3.10
    ports: ["19530:19530"]

volumes:
  pgdata:
```

### Sau khi tích hợp

```bash
# 1. Đăng ký
sc project add C:\Projects\Rag_Service --name rag_single_agent

# 2. Tạo .env
sc project env rag_single_agent -o C:\Projects\Rag_Service\.env
```

`docker-compose.yml` mới:

```yaml
services:
  app:
    build: .
    container_name: rag-app
    ports:
      - "8000:8000"
    env_file: .env
    deploy:
      resources:
        limits:
          memory: 2048M
          cpus: "1.0"
        reservations:
          memory: 256M
          cpus: "0.25"
    networks:
      - infra-net

networks:
  infra-net:
    external: true
```

```bash
# 3. Khởi động
sc project up rag_single_agent
```

Đơn giản và sạch sẽ hơn nhiều.

## 6. Rollback (nếu cần quay lại)

Nếu cần quay về cách cũ (infra riêng cho từng project):

1. Dừng project: `sc project down my_project`
2. Khôi phục `docker-compose.yml` cũ (từ git)
3. Khôi phục `.env` cũ
4. Xóa khỏi registry: `sc project remove my_project`
5. Chạy bình thường: `cd MyProject && docker compose up -d`

Data trong shared infra vẫn còn, không bị mất.
